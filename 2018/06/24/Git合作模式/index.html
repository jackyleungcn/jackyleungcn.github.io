GIT分支模型

GIT COMMIT规范

  1. head: type(scope): subject
    • type 类型
      • feat：新功能（feature）
      • fix：修补bug
      • optimize：优化
      • docs：文档（documentation）
      • style： 格式（不影响代码运行的变动）
      • refactor：重构（即不是新增功能，也不是修改bug的代码变动）
      • publish 发布
      • test：增加测试
      • chore：构建过程或辅助工具的变动
    • scope 修改范围（可选）
      • 主要是这次修改涉及到的部分，最好简单的概括
    • subject 修改的副标题
      • 主要是具体修改的加点
  2. body（可选，建议72个字符以内）
    • 修改的主体标注
  3. footer（可选）
    • 主要放置不兼容变更
    • Issue关闭的信息

GIT Merge Request规范

Merge Request至少包含3要素

1、标题，[功能名，BUG修复，Core Review修复…]

2、内容，[功能任务链接，说明…]

3、标签，根据内容选择合适的标签，若无可创建新标签，以便追溯历史纪录
